# Security policy

## Supported versions

Security fixes target the latest published minor version.

## Reporting

Do not open a public issue for a vulnerability or exposed credential. Use GitHub private
vulnerability reporting for this repository. Include affected versions, reproduction steps,
impact, and any suggested mitigation.

The maintainer will acknowledge a report within seven days and coordinate disclosure after a fix
is available. This is a best-effort open-source project and does not provide an SLA.
