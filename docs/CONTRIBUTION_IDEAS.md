# Substantive contribution ideas

These are starting points for real engineering work, not a request for metric-only pull requests.

1. Add composition-sheet normalization with synthetic fixtures for multiple observed layouts.
2. Add a structured validation summary grouped by error code and source row.
3. Add a persistence-adapter interface and an in-memory reference implementation.
4. Benchmark parsing of large synthetic monthly files and document memory usage.
5. Add examples for Express, Next.js, and command-line scripts without coupling the core package.
6. Review BDI documentation against public procurement references and propose sourced clarifications.

Open an issue before implementing a large item so scope and public API compatibility can be agreed.
